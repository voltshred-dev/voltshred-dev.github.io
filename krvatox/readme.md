# Krvatox Hardware Monitor — Web Infrastructure

Репозиторий содержит файлы веб-интерфейса и информационной базы для утилиты мониторинга аппаратных компонентов персонального компьютера **Krvatox Monitoring Program**. 

Инфраструктура спроектирована по принципу статического веб-приложения (SPA/Static Pages) и оптимизирована для хостинга на платформах вроде **GitHub Pages**. Весь интерфейс выполнен в строгом индустриальном монохромном стиле с абсолютным отсутствием скруглений элементов (90-градусные углы).

## 🚀 Архитектура и распределение языков

Проект поддерживает **8 ключевых языков**: Русский (`ru`), English (`en`), Français (`fr`), Deutsch (`de`), 中文 (`zh`), Español (`es`), 한국어 (`ko`), 日本語 (`ja`).

Логика маршрутизации распределена следующим образом:

1. **`index.html` (Корневой шлюз)** — не содержит видимого интерфейса. Выполняет роль мгновенного SEO-перенаправления. Скрипт считывает системный локальный язык браузера через `navigator.language` и перенаправляет пользователя на соответствующую изолированную страницу (например, `index_ru.html` или `index_en.html`). Если язык не поддерживается, по умолчанию отдается английская версия.
2. **`license.html` (Лицензионное соглашение EULA)** — единый файл, содержащий юридические документы на всех 8 языках внутри семантических тегов `<article>`. Определение языка браузера происходит динамически на стороне клиента (без участия пользователя), скрывая ненужные локализации.
3. **`404.html` (Страница ошибок)** — кастомная страница обработки несуществующих URL-адресов. Автоматически переводит сообщения об ошибке и навигационные элементы на язык открытого браузера и содержит интерактивный радиальный фон, реагирующий на движение курсора мыши.

## 🔍 Поисковая оптимизация (SEO) и Индексация

Для того чтобы поисковые роботы (Google, Яндекс, Bing) корректно индексировали все языковые версии, несмотря на использование клиентских JavaScript-редиректов, в архитектуру внедрены следующие решения:

* **Теги `hreflang`:** В заголовках `<head>` файлов `index.html` и `license.html` прописаны перекрестные ссылки `<link rel="alternate" hreflang="..." href="..." />`. Это связывает мультиязычные страницы в единую сеть, указывая роботам точные адреса локализаций.
* **Атрибут `x-default`:** Направляет поисковые системы на дефолтную (корневую) страницу для неопределенных регионов.
* **Семантика и атрибуты локали:** Каждый текстовый блок обернут в теги с явным указанием атрибута языка (`lang="ru"`, `lang="en"`), что исключает ошибки определения кодировок и содержимого поисковыми краулерами.
* **Запрет индексации битых ссылок:** Для `404.html` жестко задан параметр `<meta name="robots" content="noindex, follow">`, запрещающий сохранять в индекс несуществующие адреса, но позволяющий роботу пройти по ссылке «Назад» на главную страницу.
